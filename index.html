Zenvisage

Zenvisage is a data visualization tool that automatically identifies and recommends visualizations that match desired user patterns. Zenvisage offers several querying modalities for expressing visual queries as shown below.

Sketching a desired pattern as a query, and searching for similar visualizations.


Dragging and dropping a visualization from the result or overview panel, and searching for similar visualizations.


Loading the raw x,y data points depicting a pattern or trend, and searching for matching visualizations.


This function allows you to specify a query pattern using an equation, e.g. y=sin(0.5*x).


Smoothing tries to capture important features in the data while alleviating the noise. Users can interactively tune the smoothing algorithm and parameters to visualize how various smoothing parameters impact query results.


Zenvisage offers two functionalities for dynamically faceting and comparisons between different subsets of the data through filtering and dynamic classes. Filtering allows users to submit SQL-like predicates as filter constraints. The dynamic class feature allows users to bucket data points into customized classes based on existing properties to compare between the customized classes.


Entity Search

The Entity Search project provides a framework for building an entity-enabled Lucene searchable index. The users can submit two types of queries which will be handled by the enhanced index. The first type is the window pattern, eg. {uiuc cs #person}. This means uiuc, cs AND entity person should appear in the same document. The second pattern is the disjunction pattern, eg. [(uiuc | cs) #person]<10>. This means uiuc OR cs should appear within 10 words window of entity person. Both types of queries return "person" objects instead of websites which contain the "person." Thus, the granularity of search results have shrunk from documents to objects.

Noisy Image Recovery

This is a machine learning project for recovering noisy MNIST images using Boltz-mann machine model and mean field inference.The mean field inference is implemented from scratch. After preprocessing of the data, the de-noise function takes in a noisy image and reconstructs the original image using mean field inference. Running the algorithm on different theta values, the best accuracy of reconstruction was 99.3% while the worst reconstruction accuracy was 92%. Overall, the total accuracy was 96.7 %. The output seemed to be as expected because noisy pixels seemed to have ‘smoothed’ out. So the random pixels of white among black values were gone. Web Crawler

A web crawler implemented in python which scrapes Wikipedia to store information about actors and movies into a graph data structure. A graph library was designed to convert the scraped data into a graph. Then the relavent information was converted to JSON for storage. Finally, basic query functions were implemented to answer basic questions about the data. For example, one could find out which age group earned the most money based on the data.
